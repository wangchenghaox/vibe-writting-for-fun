# 大纲生成

把题材、人物目标和冲突压力整理成可执行的创作蓝图。此 skill 同时服务主 Agent 的任务编排和子 Agent 的大纲产出。

## 适用场景

- 用户要求总纲、卷纲、章节细纲、剧情规划、故事线或设定框架。
- 用户要求整理已有构思，或把零散设定变成可写的大纲。
- 不适用于直接写完整章节；写章节交给 chapter-writer。

## 角色边界

- 主 Agent：只确认需求、整理约束、列简短计划，并调用 `create_sub_agent` 委派；不要直接输出完整大纲、世界观、细纲或调用保存工具。
- 子 Agent：执行任务单，读取资料、生成大纲、按要求保存；不能向用户提问或等待输入，信息不足时把缺口和建议下一步写回执行结果。
- 主 Agent 委派时，`task` 必须写清：大纲类型、故事规模、题材基调、主角目标、核心冲突、必须读取的文件、输出结构、是否保存、目标路径、覆盖是否已确认、验收标准。
- 如果用户需求仍开放，先使用 requirement-confirmer 收束方向，再委派。

## 上下文读取

- 正式整理、生成、细化或保存大纲前，先按 progress-summarizer 读取小说根目录的 `progress.md`，确认当前进度、已完成内容和待确认问题。
- 如果 `progress.md` 不存在，只做轻量目录检查，不要逐个读取所有文件；大纲生成、修改或保存后必须更新进度总结。
- 主 Agent 委派任务单应写清 `progress.md` 路径；子 Agent 返回结果时说明是否已更新进度总结。
- 文件路径使用当前 sandbox 相对路径，不要加 `novels/` 前缀。
- 如用户提到已有小说，优先定位 `{novel_slug}/` 下的 Markdown 文件。
- 如用户要求续接或细化已有大纲，使用 `search_files` 或 `list_files` 查找 `{novel_slug}/outlines/*.md`，再用 `read_file` 读取。
- 不确定要细化哪个大纲时，子 Agent 不猜测覆盖，返回需要主 Agent 确认的候选路径。

## 工作流程

1. 明确题材、主角目标、核心冲突、爽点或悬疑点，以及故事预期规模。
2. 先搭总纲：世界规则、主线目标、阶段压力、主要对手、主题情绪。
3. 再拆卷纲：每卷写清阶段目标、局势变化、关键转折、人物关系变化。
4. 细化章节细纲时，每章必须包含目标、冲突、反转、信息增量和结尾钩子。
5. 检查主线目标是否持续可见，避免只有设定展示没有行动压力。
6. 设计完成后，如果任务单要求先出方案，返回简短方向摘要和建议主 Agent 请用户确认的点；不要直接向用户提问。
7. 如果任务单已明确继续扩写、细化或保存，子 Agent 直接完成。
8. 如果任务单或用户确认保存，使用 `write_file` 或 `edit_file` 写入 Markdown 文件。
9. 修改或生成新内容后，更新 `progress.md`，记录大纲状态、影响文件、待确认问题和下一步。

## 保存规则

- 保存大纲时只使用 `write_file` 或 `edit_file`，不要使用领域保存工具。
- 默认写入 `{novel_slug}/outlines/{outline_id}.md`，这是当前 sandbox 相对路径，不要加 `novels/` 前缀。
- `outline_id` 未指定时可建议 `main`、`volume_01` 或 `chapters_01_10`，交回主 Agent 确认。
- 文件类型必须是 Markdown，扩展名必须是 `.md`。
- `content` 必须是完整 Markdown 大纲文本，不要只保存一句说明。
- 如果当前任务没有完整大纲，先生成大纲；无法生成时返回缺口，不要空保存。
- 用户确认前不要保存，也不要覆盖已有 outline。

## Markdown 固定格式

```markdown
---
type: outline
id: {outline_id}
scope: total|volume|chapters
status: draft
---

# 大纲：{title}

## 核心设定

## 主线目标

## 阶段规划

## 关键转折

## 角色变化
```

## 输出要求

- 总纲按“核心设定、主线、阶段目标、关键转折、角色变化”组织。
- 卷纲按卷输出，每卷包含目标、主要冲突、结尾变化。
- 章节细纲用列表输出，每章包含目标、冲突、反转和钩子。
- 如需主 Agent 再确认，结尾写“建议主 Agent 确认：...”而不是直接问用户。

## 质量标准

- 主线目标要持续可见。
- 每个阶段都要推动关系、信息或局势变化。
- 避免只有世界观介绍，没有行动压力。
- 大纲要能直接指导章节写作，不要停留在抽象概念。

## 常见误区

- 主 Agent 读取本 skill 后直接写出完整大纲或世界观。
- 把世界观资料堆成百科，没有主角行动线。
- 章节细纲只写事件标题，没有冲突、反转或结尾钩子。
- 未读取已有大纲就声称已经延续。
- 用户没有明确保存时擅自覆盖已有 outline。
- 保存时没有传完整 `content`。
