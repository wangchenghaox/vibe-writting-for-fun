# 章节写作

把大纲、前文和用户目标转化为完整可读的章节正文。此 skill 同时服务主 Agent 的任务编排和子 Agent 的实际写作。

## 适用场景

- 用户要求写、生成、创作、起草、续写或改写某一章。
- 用户要求保存某章内容，且已有完整章节正文或明确要求子 Agent 生成后保存。
- 不适用于只做人物设定、总纲规划或审稿；这些任务交给对应 skill。

## 角色边界

- 主 Agent：只确认需求、整理约束、列简短计划，并调用 `create_sub_agent` 委派；不要直接输出完整章节、改写稿或调用保存工具。
- 子 Agent：执行主 Agent 的任务单，读取上下文、生成正文、按要求保存；不能向用户提问或等待输入，信息不足时把缺口和建议下一步写回执行结果。
- 主 Agent 委派时，`task` 必须写清：章节编号/标题、目标字数或篇幅、视角人物、剧情承接、必须读取的文件、是否保存、目标路径、覆盖是否已确认、验收标准。
- 不要把主 Agent 自己生成的大段正文塞进 `task` 让子 Agent 只负责保存。

## 上下文读取

- 正式写作、续写、改写或保存章节前，先按 progress-summarizer 读取小说根目录的 `progress.md`，确认当前进度、最近变更和下一步。
- 如果 `progress.md` 不存在，只做轻量目录检查，不要逐个读取所有文件；章节生成、修改或保存后必须更新进度总结。
- 主 Agent 委派任务单应写清 `progress.md` 路径；子 Agent 返回结果时说明是否已更新进度总结。
- 文件路径使用当前 sandbox 相对路径，不要加 `novels/` 前缀。
- 如用户指定大纲，使用 `search_files` 或 `list_files` 查找 `{novel_slug}/outlines/*.md`，再用 `read_file` 读取。
- 如需要承接前文，先用 `list_files` 查看 `{novel_slug}/chapters/*.md`，再用 `read_file` 读取相邻章节。
- 未找到前文或大纲时可以继续写，但要在执行结果中说明依据来自当前对话和任务单。

## 工作流程

1. 明确章节编号、标题、视角人物、章节目标、核心冲突和结尾钩子。
2. 读取必要的大纲和相邻章节，提取上一章结尾状态、未解决问题和人物当下动机。
3. 构思本章场景推进：开场行动、中段阻力、信息变化、情绪转折、结尾悬念。
4. 子 Agent 生成、创作、续写或改写章节时，必须先输出完整章节正文，不要先调用保存工具。
5. 正文完成后自查一次：人物行动是否有因果，场景是否推进，结尾是否留下下一步行动。
6. 只有任务单或用户明确要求保存时，才调用 `write_file` 或 `edit_file` 写入 Markdown 文件。
7. 修改或生成新内容后，更新 `progress.md`，记录章节状态、影响文件、待确认问题和下一步。

## 保存规则

- 保存章节时只使用 `write_file` 或 `edit_file`，不要使用领域保存工具。
- 新章节默认写入 `{novel_slug}/chapters/{chapter_id}.md`，这是当前 sandbox 相对路径，不要加 `novels/` 前缀。
- 覆盖已有章节时先用 `read_file` 读取原文，再用 `edit_file` 替换；新建章节用 `write_file`。
- 目标文件已存在但任务单未说明允许覆盖时，不要覆盖，返回“需要主 Agent 确认覆盖”。
- 文件类型必须是 Markdown，扩展名必须是 `.md`。
- `content` 必须传完整 Markdown 文件内容，不要传摘要、说明文字或空字符串。
- 如果没有完整正文，先生成正文；无法生成时返回缺口，不要空保存。

## Markdown 固定格式

```markdown
---
type: chapter
id: {chapter_id}
title: {chapter_title}
status: draft
---

# {chapter_title}

正文内容从这里开始。
```

## 输出要求

- 子 Agent 返回结果时先给章节标题，再给正文或保存结果。
- 正文应直接进入场景，不要用“下面是第二章”之类的长说明占篇幅。
- 如只要求保存，简短说明保存路径；如保存失败，说明缺少什么内容。
- 保存前确保生成或确认的正文与写入 Markdown 完全一致。

## 质量标准

- 开头尽快进入具体场景。
- 每一段都服务于动作、信息、情绪或冲突。
- 结尾留下下一步行动或悬念。
- 不要擅自覆盖已有章节，除非任务单明确说明用户已确认。
- 玄幻、修仙、悬疑、都市等题材要保持既有氛围，不要突然换文风。

## 常见误区

- 主 Agent 读取本 skill 后直接写出完整章节。
- 只说“我来保存”却没有生成正文，导致 `content` 缺失。
- 还没形成完整章节正文就调用保存工具。
- 只堆世界观、功法和设定，没有动作、冲突或人物选择。
- 未读取前文就续写，造成上一章状态断裂。
- 用户没有明确覆盖时重写已有章节。
